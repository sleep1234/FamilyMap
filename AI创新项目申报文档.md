## 一、项目背景

### 1.1 项目初衷

本项目的开发源于一个简单而温暖的想法——**家庭成员之间的相互照看与守护**。

在快节奏的工作生活中，家人之间的关心往往停留在电话问候的层面。特别是对于年长的家人，我们常常担心他们的安全，却又无法时刻陪伴在身边。如何让家人之间的关怀更加及时、更加具体，成为本项目的核心出发点。

**一个真实的应用场景**：每天下班回家时，父母可以通过平台实时查看我的定位信息，根据我的位置和距离来合理安排做饭时间。当我快到家时，饭菜刚好出锅；当我要加班时，他们可以提前知晓。这种基于位置信息的生活协调，让家庭成员之间的配合更加默契，也让日常生活多了一份温馨。

### 1.2 行业痛点

现有的位置共享解决方案存在诸多问题，难以满足家庭用户的真实需求：

- **安装门槛高**：传统位置共享App需要下载安装，对于老年人、儿童等数字弱势群体使用困难
- **隐私安全担忧**：商业位置共享App通常将数据上传至云端，用户对个人位置隐私存在担忧
- **功能单一**：现有解决方案多为简单的位置显示，缺乏智能化的场景感知和异常预警能力
- **成本较高**：类似功能的商业App（Life360/Jagat）年费约100-300元/人
- **互动性不足**：远程家人之间缺乏情感连接，位置共享仅停留在"知道在哪"的层面

### 1.3 创新契机

边缘计算和智能算法的发展为家庭位置共享提供了新的解决方案。特别是轻量化AI推理技术的成熟，使得在边缘设备上实现智能行为分析成为可能。本项目正是基于这些技术突破，将AI智能化理念融入位置共享全流程，打造零安装、隐私安全、智能感知的家庭位置共享平台。

## 二、项目应用实现方式

### 2.1 技术架构

本平台采用前后端分离架构，前端基于Flutter构建跨平台移动应用，后端采用Node.js + Express框架，整体架构分为三层：

```
┌──────────────────────────────────────────────────┐
│                    前端展示层                       │
│  Flutter跨平台 │ 高德地图SDK │ WebSocket实时通信  │
│  (iOS + Android双端原生体验)                       │
└────────────────────┬─────────────────────────────┘
                     │ HTTP REST API + WebSocket
┌────────────────────┴─────────────────────────────┐
│              Express 业务逻辑层                     │
│  位置服务 │ 围栏检测 │ 异常预警 │ 聊天互动        │
└────────────────────┬─────────────────────────────┘
                     │
┌────────────────────┴─────────────────────────────┐
│             智能算法引擎层                          │
│  停留检测 │ 异常检测 │ 场所推断 │ 驾驶评分        │
└────────────────────┬─────────────────────────────┘
                     │
┌────────────────────┴─────────────────────────────┐
│                  数据持久层                         │
│           SQLite (轻量级 + 零部署成本)             │
└──────────────────────────────────────────────────┘
```

### 2.2 核心AI应用：智能行为分析

**（1）智能停留检测与场所推断**

系统实现了基于规则引擎的智能停留检测和场所推断算法：

- 基于GPS速度判断静止/移动（阈值：3.0 m/s）
- 漂移容差（200m）避免小范围移动误判
- 离开确认机制：连续2次确认点位移才判定真正离开
- 最小停留通知时长：10分钟以下的停留不发通知，避免路口等红灯误报
- 场所类型自动推断：通过地址关键词正则匹配，自动识别home/work/school/food/fun/hospital

**创新价值**：传统位置App只提供原始坐标，本系统将坐标转化为有意义的语义信息（"张三到家了"、"李四在公司"、"王五离开了学校"），极大提升了信息可读性和用户体验。

**（2）智能异常检测与安全预警**

- **碰撞/事故风险检测**：维护9个点位的滑动窗口速度数据，计算窗口均值速度，检测高速移动（>60 km/h）和急刹车（速度在30秒内从>20 m/s骤降至<3 m/s）
- **低电量预警**：电量<15%且未充电时，每小时推送一次通知
- **长时间未活跃检测**：>24小时未更新位置则发出"失联预警"

**（3）智能地理围栏**

基于卡尔曼滤波思想的轻量实现，需连续3次GPS点位确认状态变化才触发事件，有效去除GPS定位噪声，防止误报。

**（4）驾驶行为评分**

多维特征提取：超速比例、急刹车比例、高速比例、平均速度，综合评分公式：
```
score = 100 - speedingRate*200 - hardBrakeRate*500 - (avgSpeed-15)*1.5
```
四级评价：A(≥90) / B(≥75) / C(≥60) / D(<60)

**（5）隐私保护AI**

- 模糊位置：对GPS坐标添加随机偏移（±0.005度，约500米）
- 幽灵模式：invisible模式完全不出现在他人地图上；blur模式仅发送模糊坐标
- 自部署架构：所有数据存储在用户自有服务器的SQLite中，无第三方数据泄露风险

### 2.3 中国电信星辰超级智能体（TeleAgent）在开发中的应用

本项目在开发过程中深度借助了中国电信星辰超级智能体（TeleAgent）作为AI开发辅助工具，显著提升了开发效率和代码质量：

**（1）智能代码优化**

- **代码审查与重构**：利用星辰智能体的代码理解能力，对复杂算法（如停留检测、地理围栏、异常检测）进行深度分析和优化建议
- **性能优化**：通过AI分析代码瓶颈，优化数据库查询、内存使用和算法复杂度
- **安全加固**：借助AI识别潜在安全漏洞，完善认证机制、输入验证和数据加密

**（2）抓包数据分析**

- **API调试**：利用星辰智能体分析高德地图API、Bark推送服务等外部接口的请求/响应数据
- **协议解析**：通过AI辅助理解WebSocket通信协议，优化实时位置同步机制
- **异常诊断**：借助AI分析网络请求异常，快速定位和解决接口调用问题

**（3）技术方案设计**

- **架构设计**：通过与AI对话讨论，确定了Node.js + Express + Socket.IO + SQLite的技术选型
- **算法实现**：利用AI辅助设计停留检测、地理围栏、异常检测等核心算法
- **文档生成**：借助AI快速生成技术文档和注释，提升代码可维护性
